# GoPay 支付网关管理平台 - PRD

## 1. 产品概述

### 1.1 产品定位

面向支付服务商的网关管理后台，用于管理少量合作商户的支付通道接入、订单交易、进件审核和对账等核心业务。

### 1.2 目标用户

- **管理员**：平台拥有者，拥有全部操作权限
- **运营人员**：日常运营，仅拥有查看权限，不可修改配置

### 1.3 支付通道

- 支付宝（基于 gopay SDK，V3 API）
- 微信支付（基于 gopay SDK，V3 API）

### 1.4 技术背景

- 前端：Vue 3 + TypeScript + Vite + Vuetify 3（Material Design）+ ECharts
- 后端：Go + Gin + GORM（项目 gopay-platform）
- 支付 SDK：gopay（支持支付宝、微信等多通道）
- 设计风格：Material Design 3 + 渐变点缀，Soft Blue (#3B9EFF → #7BBFFF) 主色调，白色侧边栏

## 2. 功能模块

平台共 **8 个模块**，以下逐一说明。

---

### 2.1 仪表盘

> 登录后首页，展示平台核心经营数据。

| 功能项 | 说明 |
|--------|------|
| 今日交易额卡片 | 展示当日累计交易金额 |
| 今日交易笔数卡片 | 展示当日累计交易笔数 |
| 今日成功率卡片 | 成功订单数 / 总订单数 |
| 通道分布饼图 | 按支付宝/微信展示交易金额占比 |
| 近7日交易趋势图 | 折线图，交易额 + 笔数双轴展示 |
| 快捷入口 | 待审核进件数、待处理退款数，点击跳转对应页面 |

---

### 2.2 商户管理

> 管理合作商户的基本信息和应用配置。

#### 2.2.1 商户列表

| 功能项 | 说明 |
|--------|------|
| 商户列表 | 展示商户名称、联系人、联系方式、状态、创建时间 |
| 搜索筛选 | 按商户名称、状态筛选 |
| 新增商户 | 填写商户名称、联系人、联系方式等基本信息 |
| 编辑商户 | 修改商户基本信息 |
| 启用/停用 | 切换商户状态，停用后其下所有应用停止交易 |

#### 2.2.2 商户应用

| 功能项 | 说明 |
|--------|------|
| 应用列表 | 展示商户下所有应用，包含 AppID、应用名称、状态 |
| 新增应用 | 填写应用名称，自动生成 AppID 和 AppSecret |
| 应用配置 | 配置回调地址（return_url、notify_url） |
| 绑定通道 | 选择该应用启用的支付通道（支付宝/微信） |
| 启用/停用 | 切换应用状态 |

---

### 2.3 进件管理

> 代替商户向支付宝/微信提交资质材料，完成支付通道开通。

#### 2.3.1 进件申请

| 功能项 | 说明 |
|--------|------|
| 进件列表 | 商户名称、申请通道、状态、提交时间 |
| 状态流转 | 待提交 → 待审核 → 审核中 → 已通过 / 已驳回 |
| 新建进件 | 选择商户、选择通道（支付宝/微信），填写资质材料 |
| 资质材料 | 营业执照、法人身份证、银行账户信息、行业类目等 |
| 提交进件 | 将材料通过 API 提交到对应支付通道 |
| 审核操作 | 查看通道返回的审核结果，通过/驳回 |
| 同步状态 | 主动查询通道侧最新审核状态 |

#### 2.3.2 进件记录

| 功能项 | 说明 |
|--------|------|
| 操作日志 | 每次提交、审核、状态变更的时间线记录 |
| 通道反馈 | 保存通道侧返回的原始审核意见 |

---

### 2.4 支付配置

> 管理平台级别的支付通道密钥和证书。

| 功能项 | 说明 |
|--------|------|
| 通道列表 | 支付宝、微信两个通道卡片，展示启用状态 |
| 支付宝配置 | AppID、应用私钥、应用公钥证书、支付宝公钥证书、支付宝根证书 |
| 微信支付配置 | AppID、商户号(MchID)、API V3 密钥、商户证书序列号、商户私钥 |
| 启用/停用 | 通道级别的开关 |
| 连通性测试 | 一键测试当前配置是否能正常调用通道 API |

---

### 2.5 订单中心

> 管理所有支付、退款、转账订单。

#### 2.5.1 支付订单

| 功能项 | 说明 |
|--------|------|
| 订单列表 | 订单号、商户名称、应用名称、金额、通道、状态、创建时间 |
| 状态 | 待支付、已支付、已关闭 |
| 筛选 | 按商户、通道、状态、时间范围筛选 |
| 订单详情 | 完整支付信息、通道返回信息、操作时间线 |
| 关单操作 | 对待支付订单执行关单 |
| 同步状态 | 主动向通道查询最新订单状态 |

#### 2.5.2 退款订单

| 功能项 | 说明 |
|--------|------|
| 退款列表 | 退款单号、关联支付单号、退款金额、状态、时间 |
| 状态 | 处理中、已退款、退款失败 |
| 发起退款 | 选择支付订单，填写退款金额和原因 |
| 查询退款 | 向通道查询退款处理进度 |

#### 2.5.3 转账订单

| 功能项 | 说明 |
|--------|------|
| 转账列表 | 转账单号、收款方、金额、通道、状态、时间 |
| 状态 | 处理中、已到账、转账失败 |
| 发起转账 | 选择通道、填写收款方信息和金额 |
| 查询状态 | 向通道查询转账处理进度 |

---

### 2.6 交易记录

> 查看所有交易的完整流水和通道回调。

#### 2.6.1 交易流水

| 功能项 | 说明 |
|--------|------|
| 流水列表 | 时间、流水号、关联订单号、类型（支付/退款/转账/关单）、金额、状态 |
| 筛选 | 按类型、通道、时间范围筛选 |
| 流水详情 | 完整的请求/响应数据 |

#### 2.6.2 回调记录

| 功能项 | 说明 |
|--------|------|
| 回调列表 | 时间、通道、关联订单号、回调类型、处理结果（成功/失败） |

| 重试 | 对处理失败的回调手动重新处理 |

| 对账结果 | 匹配笔数、差异笔数、对账金额汇总 |

#### 2.7.2 差异记录

| 功能项 | 说明 |
|--------|------|
| 差异列表 | 订单号、差异类型（金额不符/状态不符/单边账）、本地金额、通道金额 |
| 差异处理 | 标记已处理、添加处理备注 |

---

## 5. 实施优先级

按依赖关系和业务价值排序：

| 阶段 | 模块 | 说明 |
|------|------|------|
| P0 | 系统管理 | 基础设施，用户和权限是其他模块的前提 |
| P0 | 支付配置 | 通道密钥配置，支付功能的前提 |
| P1 | 商户管理 | 核心业务，商户和应用是订单的前提 |
| P1 | 进件管理 | 商户开通支付通道的流程 |
| P2 | 订单中心 | 核心交易管理 |
| P2 | 交易记录 | 交易流水和回调查看 |
| P2 | 仪表盘 | 依赖订单数据，放在订单之后 |
| P3 | 对账管理 | 运营保障，优先级最低 |
